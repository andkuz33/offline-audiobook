# Синхронный переводчик — iOS (прототип)

Прототип приложения синхронного перевода речи через **Gemini Live API**.

- **Слушать окружающих** — микрофон телефона слышит речь на **английском или китайском**, Gemini автоматически определяет язык и озвучивает перевод **по-русски вам в наушник (AirPods)**.
- **Говорить** (push-to-talk) — вы говорите **по-русски**, перевод звучит на **английском или китайском** (тумблер) **из динамика телефона** — для собеседника.

> ⚠️ Это **прототип ключевой части**, а не готовый продукт. Код написан под Xcode и
> **не компилировался** в среде, где создавался — обязательно проверьте сборку и
> поведение на реальном iPhone (микрофон и Bluetooth в симуляторе работают не так).

## Что внутри

| Файл | Назначение |
|------|-----------|
| `SyncTranslatorApp.swift` | Точка входа SwiftUI |
| `ContentView.swift` | Экран: «Слушать», push-to-talk «Говорить», тумблер EN/ZH |
| `TranslatorViewModel.swift` | Оркестрация режимов, разрешения, связка аудио ↔ сеть |
| `GeminiLiveClient.swift` | WebSocket-клиент Gemini Live API (setup → аудио → перевод) |
| `AudioEngine.swift` | Захват микрофона (PCM16 16 кГц) и воспроизведение (PCM16 24 кГц), эхоподавление |
| `Config.swift` | Модель, голос, чтение API-ключа |
| `AppModels.swift` | Языки, режимы, system-инструкции для перевода |

## Как собрать

1. На Mac в **Xcode**: `File → New → Project → iOS → App`, интерфейс **SwiftUI**, язык **Swift**. Назовите, например, `SyncTranslator`.
2. Удалите сгенерированные `ContentView.swift` и `…App.swift` и перетащите в проект все файлы из папки `Sources/` (галочка *Copy items if needed*).
3. **API-ключ.** Получите ключ в [Google AI Studio](https://aistudio.google.com/apikey). В `Info.plist` добавьте строковый ключ `GEMINI_API_KEY` со значением ключа. (Лучше — через `.xcconfig`/Build Setting, чтобы не коммитить ключ в репозиторий.)
4. **Разрешения и фоновый звук** — добавьте в `Info.plist`:

   ```xml
   <key>NSMicrophoneUsageDescription</key>
   <string>Приложению нужен микрофон, чтобы слышать речь и переводить её.</string>

   <key>UIBackgroundModes</key>
   <array>
     <string>audio</string>
   </array>
   ```

5. Запустите на **реальном iPhone** (не симуляторе). Вставьте AirPods, нажмите «Слушать окружающих».

## Как пользоваться

- **Переговоры, слушаю собеседников:** вставьте AirPods, телефон положите на стол (его микрофон лучше слышит комнату, чем микрофон AirPods). Нажмите **«Слушать окружающих»** — английская/китайская речь будет звучать по-русски вам в ухо.
- **Хочу ответить:** **зажмите** кнопку микрофона, выберите язык тумблером **EN/ZH**, говорите по-русски — перевод прозвучит из динамика телефона собеседнику. Отпустили — режим выключился.

## Важные технические замечания

- **Задержка ~1–3 c.** Это «почти синхронно»: модели нужно услышать кусок фразы. Настоящий синхрон с опережением пока недостижим.
- **Эхо.** Включён `voiceChat`-режим аудиосессии и `setVoiceProcessingEnabled` — аппаратное эхоподавление, чтобы звучащий перевод не попадал обратно в микрофон. Push-to-talk дополнительно разводит режимы во времени.
- **Модель.** По умолчанию `gemini-2.0-flash-live-001` (стабильный Live-вариант). В `Config.swift` можно переключить на 2.5-live-preview. Модели «Gemini 3.5 Flash» не существует.
- **Аудиоформаты Live API:** вход — PCM16 16 кГц моно; выход — PCM16 24 кГц моно. Конвертация делается в `AudioEngine` через `AVAudioConverter`.

## Что ещё стоит доделать (за рамками прототипа)

- Авто-VAD вместо push-to-talk для исходящего перевода.
- Показ субтитров (текстовая модальность параллельно с аудио).
- Автоопределение «языка собеседника» вместо ручного тумблера.
- Полноценный фоновый режим и обработка обрывов сети с переподключением.
- Раздельные устройства для двусторонних переговоров.
